About This Application

The **Movie Collection App** is a full-stack web application designed to demonstrate robust user authentication, role-based access control, and seamless interaction with a database and a third-party API. It serves as a personal portfolio piece showcasing my abilities in building secure, dynamic, and user-friendly web experiences.

Core Features:

  • User Authentication: Secure user registration and login using industry-standard practices (JWT for session management, bcrypt for password hashing).
  • Role-Based Access Control (RBAC): Implemented a tiered permission system with 'user', 'admin', and 'super_admin' roles, restricting access to certain features and API endpoints based on the user's role.
    • User: Can manage their own movie collection.
    • Admin: Can add and edit any movies in the collection.
    • Super Admin: Has full CRUD (Create, Read, Update, Delete) capabilities over all movies.
  • Personal Movie Collection Management: Users can add, view, edit, search, and sort their private movie entries.
  • Public Trending Movies: Integration with The Movie Database (TMDB) API to display trending movies on the homepage for unauthenticated visitors, enhancing discoverability and user engagement. This integration utilizes a backend proxy for API key security and controlled data fetching.
  • Responsive Design: Built with a focus on usability across various devices.

Technologies Used:

Backend:

  • Node.js: JavaScript runtime environment.
  • Express.js: Web application framework for Node.js, for building RESTful APIs.
  • MongoDB: NoSQL database for flexible data storage.
  • Mongoose: ODM (Object Data Modeling) library for MongoDB and Node.js.
  • JSON Web Tokens (JWT): For secure authentication and authorization.
  • Bcrypt.js: For password hashing.
  • Node-fetch: For making HTTP requests to external APIs (TMDB).
  • dotenv: For managing environment variables securely.

Frontend:

  • HTML5: Structure of the web pages.
  • CSS3: Styling and layout.
  • Vanilla JavaScript: Client-side logic and dynamic content updates.

Motivation & Learning Outcomes:

This project was undertaken to solidify my understanding of full-stack development principles. Key learning outcomes include:

  • Designing and implementing a secure authentication and authorization system.
  • Building robust RESTful APIs with input validation and error handling.
  • Integrating and securely managing third-party API data.
  • Creating a dynamic and responsive user interface that interacts seamlessly with a backend.
  • Practicing deployment strategies for full-stack applications.
